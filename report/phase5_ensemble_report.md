## 3. 加权 Blend

### 3.1 搜索结果

用 `scipy.optimize.minimize（Nelder-Mead）` 从多个初始点出发，在 OOF 上搜索最优权重（th=0.50）：

| 模型 | 权重 |
|------|------|
| LGB  | 0.054 |
| XGB  | **0.893** |
| CAT  | 0.054 |

XGB 权重远超预期（接近 0.9）。原因：优化目标是 th=0.50 时的准确率，而 XGB 的 OOF 概率整体偏低（最优阈值 0.47），在 th=0.50 时恰好落在更多正确预测的一侧，因此被优化器赋予高权重。

### 3.2 Blend 结果

|--|---------|---------|--------|
| Blend | 0.81617 | 0.4575 | 0.81732 |

## 4. LR Stacking

以三组 OOF 概率为特征，训练 `LogisticRegression(C=1.0)` 元模型（OOF 预测已为 out-of-fold，直接训练不引入泄漏）：

| 模型 | 系数 |
|------|------|
| LGB  | 0.123 |
| XGB  | **1.500** |
| CAT  | 0.556 |

系数分布与 Blend 权重一致——XGB 在元模型中权重最高，反映 XGB 的概率分布为最终预测贡献了最多的区分力。

| | th=0.50 | 最优阈值 | th=最优 |
|--|---------|---------|--------|

## 7. 关键发现

1. **阈值下移是本阶段最大收益来源**：三个模型最优阈值均低于 0.5（0.455–0.48），单纯阈值优化比 th=0.50 提升 0.00046–0.00196。概率偏保守可能源于类别不平衡（正类 50.4%）或模型 bagging 对极端概率的平滑效应。

2. **XGB 在集成中贡献最大**：尽管 XGB CV Mean（0.81266）是三个模型中最低的，但其 OOF 概率分布与阈值的配合最好，在 Blend 和 LR Stack 中均被赋予最高权重/系数。CV Mean 低并不意味着集成价值低。

3. **LR Stack 小幅优于简单 Blend**（0.81836 vs 0.81732）：LR 能学习到非对称的模型权重，而非线性混合，额外提升约 +0.001。

## 9. Phase 6 建议

当前 OOF acc=0.81836 是纯数据驱动的集成结果。下一步可考虑：

1. **提交并观察 CV/LB gap**：先提交当前文件，获得真实榜单分数，判断是否需要进一步优化
2. **业务规则后处理**（需在 OOF 上验证提升 ≥ 0.001 才采用）：
   - CryoSleep=True 且预测概率 > 0.4 → 强制 True
   - 组内一致性修正（组内预测均值 > 0.7 且某成员 < 0.5 → 修正）
3. **特征增强后重训**：
   - 目标编码 `LastName_TE`、`Cabin_TE`（在 CV 内计算）
   - 补充 `SpendPerGroupMember`、`AvgSpendPerService`
4. **GroupKFold 重评**：验证 StratifiedKFold 的 CV 分数是否存在组泄漏导致的虚高
